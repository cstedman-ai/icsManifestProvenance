# 20260325 вҖ” Slack Integration вҖ” Inventory Photo Notifications

> **Purpose:** Define how to automatically push annotated packing slip photos to a designated Slack inventory channel when a Coreweave user captures and signs a photo on their mobile device, alongside the existing device-save and server-upload flows.
>
> **Related docs:**
> - `docs/methodologies/20260325.ServerStorage.SQLiteGallery.md`
> - `docs/methodologies/20260324.MobileReceiving.ScanAndVerify.md`
> - `docs/from.ICS/20260324.GuidingPrinciples.ReceivingAndNetsuiteIntegration.md`

---

## 1. Rationale

The receiving team, logistics managers, and operations staff don't live inside the ICS Supreme app all day вҖ” they live in Slack. When a dock worker scans and signs a packing slip, the people who need to know about it should see it immediately in Slack with enough context to act on it, without opening another tool.

**Goal:** Every accepted packing slip photo triggers a Slack message to a configured `#inventory-receiving` channel containing the annotated image, the user who captured it, the PO number, vendor, and timestamp вҖ” all within seconds of the Accept tap.

---

## 2. Three-Destination Flow

When the user taps **Accept** on the annotation screen, three things happen in parallel:

```
                    в”Ңв”Җв”Җв”Җв”Җ 1. Download PNG to device (blob URL + <a download>)
                    в”Ӯ
  User taps Accept в”Җв”јв”Җв”Җв”Җв”Җ 2. Upload PNG to server (POST /api/photos вҶ’ disk + SQLite)
                    в”Ӯ
                    в””в”Җв”Җв”Җв”Җ 3. Push PNG + metadata to Slack (server вҶ’ filesUploadV2)
                              The actual image file is uploaded to the channel,
                              not a link вҖ” it renders inline for all recipients.
```

Destinations 1 and 2 already exist (or are planned per the SQLite Gallery methodology). Destination 3 is the focus of this document.

**Important:** The Slack push happens **server-side**, not from the browser. The frontend sends the photo to the Express backend, and the backend handles the Slack API call. This avoids exposing Slack tokens in client-side code and allows retries, rate-limit handling, and centralized configuration.

1. Go to [api.slack.com/apps](https://api.slack.com/apps) вҶ’ **Create New App** вҶ’ **From scratch**
2. Name: `ICS Supreme Receiving`
3. Workspace: Select the Coreweave / ICS workspace

### 3.2 Required Scopes (Bot Token)

Under **OAuth & Permissions вҶ’ Bot Token Scopes**, add:

| Scope               | Purpose                                      |
|---------------------|----------------------------------------------|
| `chat:write`        | Post messages to channels                    |
| `files:write`       | Upload images as file attachments            |
| `chat:write.public` | Post to public channels without joining them |

### 3.3 Install to Workspace

Click **Install to Workspace** and authorize. Copy the **Bot User OAuth Token** (`xoxb-...`). This token goes into the server's environment variables вҖ” never in client code.

### 3.4 Channel Setup

Create or designate a Slack channel:

- **Channel name:** `#inventory-receiving`
- **Purpose:** Automated packing slip photos from dock receiving
- Invite the bot to the channel: `/invite @ICS Supreme Receiving`

For multi-channel routing (e.g., per-vendor channels), see Section 8.

## 5. Slack Service Implementation

### 5.1 Slack Client Singleton (`server/src/services/slack.ts`)

```typescript
import { WebClient, LogLevel } from '@slack/web-api';

const token = process.env.SLACK_BOT_TOKEN;
const enabled = process.env.SLACK_ENABLED === 'true';

let client: WebClient | null = null;

export function getSlackClient(): WebClient | null {
  if (!enabled || !token) return null;
  if (!client) {
    client = new WebClient(token, { logLevel: LogLevel.WARN });
  }
  return client;
}

export function isSlackEnabled(): boolean {
  return enabled && !!token;
}
```

### 5.2 Post Photo to Slack (`server/src/services/slackNotify.ts`)

```typescript
import { getSlackClient } from './slack';
import fs from 'fs';
import path from 'path';

interface PhotoNotification {
  filePath: string;          // absolute path to the PNG on disk
  capturedBy: string;        // user email
  dateStamp: string;         // YYYYMMDD:HHMM
  poNumber?: string;
  vendor?: string;
  channelId?: string;        // override default channel
}

export async function notifySlack(photo: PhotoNotification): Promise<void> {
  const slack = getSlackClient();
  if (!slack) return;

  const channelId = photo.channelId || process.env.SLACK_CHANNEL_ID;
  if (!channelId) return;

  const poLine = photo.poNumber
    ? `*PO:* ${photo.poNumber}`
    : '_No PO linked_';
  const vendorLine = photo.vendor
    ? `*Vendor:* ${photo.vendor}`
    : '';

  const contextLines = [
    `*Received by:* ${photo.capturedBy}`,
    `*Timestamp:* ${photo.dateStamp}`,
    poLine,
    vendorLine,
  ].filter(Boolean).join('\n');

  // Upload the image file with context message
  await slack.filesUploadV2({
    channel_id: channelId,
    file: fs.createReadStream(photo.filePath),
    filename: path.basename(photo.filePath),
    title: photo.poNumber
      ? `Packing Slip вҖ” ${photo.poNumber}`
      : 'Packing Slip вҖ” Received',
    initial_comment: [
      ':package: *Packing Slip Received*',
      '',
      contextLines,
    ].join('\n'),
  });
}
```

### 5.3 Message Format in Slack

The `filesUploadV2` call above does two things in a single API request:

1. **Uploads the actual annotated packing slip PNG** as a Slack-hosted file attached to the message.
2. **Posts a context message** (`initial_comment`) with the metadata (who, when, which PO, which vendor).

**This is not a link or a URL reference.** The actual PNG bytes are uploaded to Slack's file storage via `filesUploadV2`, so it works for private deployments where the server isn't publicly accessible.

## 6. Integration with Photo Upload Route

### 6.1 Modified Upload Handler (`server/src/routes/photos.ts`)

After saving the photo to disk and inserting the metadata into SQLite, call `notifySlack`:

```typescript
import { notifySlack } from '../services/slackNotify';

router.post('/', upload.single('photo'), async (req, res) => {
  // ... existing: save file, generate thumbnail, insert into SQLite ...

  const photoRecord = {
    id,
    filePath: absoluteFilePath,
    capturedBy: req.body.capturedBy,
    dateStamp: req.body.dateStamp,
    poNumber: req.body.poNumber || null,
    vendor: req.body.vendor || null,
  };

  // Slack notification вҖ” fire-and-forget, don't block the response
  notifySlack(photoRecord).catch((err) => {
    console.error('Slack notification failed:', err.message);
    // Log failure but don't fail the upload
  });

  res.status(201).json({ id, filename, capturedBy: photoRecord.capturedBy });
});
```

**Key design decision:** The Slack call is fire-and-forget. The HTTP response returns immediately after saving the photo. If Slack is down or rate-limited, the photo is still safely stored on the server and device.

## 7. Retry and Failure Handling

### 7.1 Built-in Retries

`@slack/web-api` automatically retries on `429 Too Many Requests` with the `Retry-After` header. No custom retry logic needed for rate limits.

### 7.2 Dead Letter Logging

For persistent failures (invalid token, channel deleted, network issues), log to a `slack_failures` table:

```sql
CREATE TABLE IF NOT EXISTS slack_failures (
  id          TEXT PRIMARY KEY,
  photo_id    TEXT NOT NULL,
  channel_id  TEXT NOT NULL,
  error       TEXT NOT NULL,
  attempts    INTEGER NOT NULL DEFAULT 1,
  created_at  TEXT NOT NULL DEFAULT (datetime('now')),
  resolved_at TEXT
);
```

A scheduled job (or manual admin endpoint) can retry unresolved failures:

```
POST /api/admin/slack/retry-failures
```

### 7.3 Slack Status in Photo Metadata

Add a `slack_status` column to the `photos` table:

```sql
ALTER TABLE photos ADD COLUMN slack_status TEXT DEFAULT 'pending';
-- Values: 'pending', 'sent', 'failed', 'disabled'
```

This lets the Gallery UI show whether a photo was successfully pushed to Slack.

## 8. Multi-Channel Routing (Optional)

For organizations that want per-vendor or per-warehouse channels:

### 8.1 Channel Mapping Configuration

```typescript
// server/src/config/slackChannels.ts

const CHANNEL_MAP: Record<string, string> = {
  default: process.env.SLACK_CHANNEL_ID!,

  // Per-vendor overrides
  'Dell':       'C0123456789',
  'SuperMicro': 'C0234567890',
  'NVIDIA':     'C0345678901',
  'Opengear':   'C0456789012',
};

export function resolveChannel(vendor?: string): string {
  if (vendor && CHANNEL_MAP[vendor]) {
    return CHANNEL_MAP[vendor];
  }
  return CHANNEL_MAP.default;
}
```

### 8.2 Usage

```typescript
await notifySlack({
  ...photoRecord,
  channelId: resolveChannel(photoRecord.vendor),
});
```

This sends Dell packing slips to `#receiving-dell`, NVIDIA to `#receiving-nvidia`, etc., while everything still goes to the default `#inventory-receiving` as a catch-all.

### 8.3 Dual-Post

To post to both a vendor-specific channel and the main channel:

```typescript
const channels = [
  CHANNEL_MAP.default,
  resolveChannel(photoRecord.vendor),
];
const uniqueChannels = [...new Set(channels)];

await Promise.allSettled(
  uniqueChannels.map((ch) =>
    notifySlack({ ...photoRecord, channelId: ch })
  )
);
```

### 9.1 Block Kit Message with Actions

Instead of plain text, use Slack's Block Kit for a richer message with action buttons:

```typescript
await slack.chat.postMessage({

    {

    {

      alt_text: 'Annotated packing slip',
    },
    {
      type: 'actions',
      elements: [
        {
          type: 'button',
          text: { type: 'plain_text', text: 'View in Gallery' },
          url: `${APP_BASE_URL}/gallery/${photoId}`,
        },
        {
          type: 'button',

```

**Note:** The `image` block requires the image to be publicly accessible via URL. For internal deployments, use `filesUploadV2` (Section 5.2) instead, which doesn't require public URLs.

### 9.2 Threaded Replies

If multiple pallets arrive for the same PO, thread subsequent photos under the first message:

```typescript
// Store the Slack message timestamp (ts) in the photos table
// ALTER TABLE photos ADD COLUMN slack_ts TEXT;

const existingThread = db.prepare(
  'SELECT slack_ts FROM photos WHERE po_number = ? AND slack_ts IS NOT NULL ORDER BY created_at ASC LIMIT 1'
).get(poNumber);

const result = await slack.filesUploadV2({
  channel_id: channelId,
  thread_ts: existingThread?.slack_ts || undefined,
  file: fs.createReadStream(filePath),
  // ...
});
```

This keeps Slack tidy вҖ” one top-level message per PO with photo replies threaded underneath.

## 11. Security Considerations

| Concern                      | Mitigation                                                           |
|-----------------------------|----------------------------------------------------------------------|
| Token exposure              | `SLACK_BOT_TOKEN` in server `.env` only, never in client bundle      |
| Channel spam / abuse        | Server-side only вҖ” users can't trigger arbitrary Slack messages       |
| PII in photos               | Photos may contain names/addresses on packing slips вҖ” channel should be private and access-controlled |
| Rate limits                 | `@slack/web-api` handles `429` automatically; file uploads are ~1/min max in practice |
| Token rotation              | Slack bot tokens don't expire but can be rotated in the app settings if compromised |
