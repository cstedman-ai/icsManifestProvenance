# 20260225 ‚Äî Catalog Flow ‚Äî Vendor Inventory to Purchase Order

---

## Overview

The Catalog Flow connects three components into an end-to-end procurement pipeline:

1. **Vendor Inventory** ‚Äî Vendors stock their available products.
2. **Coreweave Catalog** ‚Äî Coreweave users browse vendor inventories and create Purchase Orders.
3. **Vendor Portal** ‚Äî Vendors receive and fulfill those Purchase Orders.

This document details the data flow, user interactions, and component responsibilities across the full lifecycle.

## 1. Vendor Inventory (`src/pages/vendors/VendorInventory.jsx`)

### Purpose

Allows vendors to declare what products they have available and in what quantities.

### Data Sources

- **Catalog items** (`src/data/cores/catalog/`) ‚Äî Pre-defined datacenter equipment mapped to each vendor by `vendorIds`. Each vendor sees only items assigned to them.
- **Custom items** ‚Äî Vendors can add products not in the standard catalog (e.g., specialty cables, accessories, services).

### State

Inventory records are stored in `state.vendorInventory` (persisted to localStorage). Each record contains:

| Field | Description |
|---|---|
| `id` | Unique identifier |
| `vendorId` | Owning vendor's ID |
| `catalogItemId` | Reference to catalog item (null for custom items) |
| `custom` | Boolean flag for vendor-added items |
| `model` | Product name (custom items only) |
| `partNumber` | Part number (custom items only) |
| `category` | Product category (custom items only) |
| `description` | Description (custom items only) |
| `estimatedCost` | Price string (custom items only) |
| `quantity` | Current stock level |
| `updatedAt` | Last modification timestamp |

### Reducer Actions

- `SET_INVENTORY_STOCK` ‚Äî Set quantity for a catalog item
- `ADD_CUSTOM_INVENTORY_ITEM` ‚Äî Add a vendor-defined custom product
- `UPDATE_CUSTOM_INVENTORY_ITEM` ‚Äî Modify a custom item's fields
- `REMOVE_INVENTORY_ITEM` ‚Äî Delete an inventory record

### Vendor Partitioning

Each vendor only sees and manages their own inventory. Filtering is by `vendorId` matching the authenticated user's `user.vendor.id`.

## 2. Coreweave Catalog (`src/pages/Catalog.jsx`)

### Purpose

Allows Coreweave procurement users to browse what vendors have stocked, select items, and issue Purchase Orders directly from available inventory.

### Flow

#### Step 1: Vendor Selection

The page opens to a grid of all registered vendors. Each card displays:
- Vendor logo/initials and brand color
- Count of stocked items (items with `quantity > 0`)

Vendors with no stocked inventory show "No stock" but are still selectable.

#### Step 2: Browse Inventory

After selecting a vendor, the user sees only items that vendor has stocked (`quantity > 0`). Items are:
- Enriched with catalog metadata (model, description, specs, estimated cost) for catalog items
- Displayed directly from inventory fields for custom items
- Grouped by category (Servers, Switches, Transceivers, Cables & Fiber, PDUs, Cabinets & Racks, Other)
- Searchable by model, description, or part number
- Collapsible by category

Each item shows its available quantity in green.

#### Step 3: Cart Management

- Clicking "Add" on an item places it in the cart with quantity 1
- +/- controls adjust quantity, capped at the vendor's available stock
- The cart panel shows all selected items with running totals
- Items can be removed individually

#### Step 4: Issue Purchase Order

The cart includes:
- **PO Number** (required) ‚Äî e.g., `PO-2026-001`
- **Notes** (optional)
- **Submit button** ‚Äî "Issue PO to [Vendor Name]"

On submit, the system dispatches `CREATE_PO` with:

```javascript
{
  type: 'CREATE_PO',
  payload: {
    poNumber: 'PO-2026-001',
    vendor: 'Dell',            // vendor.shortName
    notes: '...',
    items: [
      {
        description: 'PowerEdge XE9680',
        partNumber: 'XE9680-H100',
        quantity: 2,
        unitPrice: 350000,
        serials: [],
      },
      // ...
    ],
  },
}
```

This creates a Purchase Order with status `issued` in `state.purchaseOrders`.

#### Step 5: Confirmation

A success screen confirms the PO was created, showing the PO number, vendor name, and line item count. The user can:
- **Create Another** ‚Äî Reset the form for a new order
- **View Purchase Orders** ‚Äî Navigate to the PO list

## 3. Vendor Portal (`src/pages/vendors/VendorPortal.jsx`)

### How the PO Arrives

The Vendor Portal filters `state.purchaseOrders` by vendor name:

```javascript
const availablePOs = state.purchaseOrders.filter(
  (po) => po.vendor === vendorName && (po.status === 'issued' || po.status === 'shipped')
);
```

Purchase Orders created from the Catalog page immediately appear in the corresponding vendor's portal because they're stored in the shared `state.purchaseOrders` array with the vendor's `shortName`.

### Vendor Fulfillment

From the portal, the vendor can:
1. Select the PO
2. Set shipped quantities and serial numbers per line item
3. Enter a tracking number
4. Submit the shipment ‚Üí PO status changes to `shipped`
5. Print a QR-coded packing slip label

## 4. Catalog Data Structure (`src/data/cores/catalog/`)

Catalog items are organized into per-vendor files:

| File | Vendor | Items |
|---|---|---|
| `dell.ts` | Dell | 3 servers + 1 cabinet |
| `nVidia.ts` | NVIDIA | 5 switches |
| `mellanox.ts` | Mellanox | 4 transceivers |
| `lumaOptics.ts` | Luma Optics | 1 transceiver + 3 fiber cables |
| `uline.ts` | Uline | 1 copper cable |
| `proficium.ts` | Proficium | 1 PDU |

The barrel `index.ts` aggregates all vendor catalogs and exports:
- `catalogItems` ‚Äî Full flat array of all items
- `getCatalogForVendor(vendorId)` ‚Äî Filtered by vendor
- `categoryLabels` ‚Äî Display names for category keys

Items reference the datacenter equipment specifications from `00.References/dcSupreme/src/components/datacenterCore/`.

## 5. Full Lifecycle Example

1. **Dell vendor** logs in ‚Üí navigates to **Inventory** ‚Üí stocks 10√ó PowerEdge XE9680 (H100) and 5√ó Dell 44U Rack Cabinets.

2. **Coreweave user** logs in ‚Üí navigates to **Catalog** ‚Üí selects **Dell** ‚Üí sees 2 stocked items.

3. Coreweave user adds 3√ó XE9680 and 2√ó 44U Cabinets to cart ‚Üí enters PO number `PO-2026-042` ‚Üí clicks "Issue PO to Dell."

4. PO `PO-2026-042` is created with status `issued`, vendor `Dell`, and 2 line items.

5. **Dell vendor** logs in ‚Üí sees `PO-2026-042` in their **Shipment Portal** ‚Üí selects it ‚Üí enters shipped quantities and tracking ‚Üí submits.

6. PO status changes to `shipped`. A QR-coded packing slip is generated for printing.

7. **Coreweave mobile user** scans the QR code at the dock ‚Üí reviews items ‚Üí signs ‚Üí accepts delivery ‚Üí PO status becomes `received`.
